02 · Autonomy architecture

Guidance state machine In development — Phase 4/6

Mission phases are explicit states; transitions are conditions on detection confidence, coverage, and vehicle/energy state. Select a state to inspect it. The transition logic runs onboard — the ground link carries telemetry, never the decision.

BAYESIAN SEARCH MAP Planned · Phase 4

The search area is a grid of target-presence probabilities. A null observation updates each seen cell by Bayes' rule against the detector's probability of detection — lowering, never zeroing, it. Guidance steers toward maximum expected detection gain rather than a fixed lawnmower: no wasted passes over cleared water, graceful handling of an imperfect detector. The simulation above runs exactly this update.

CONFIDENCE GATING

A single frame is not a detection. Confidence accumulates over consecutive high-confidence frames and decays on misses; the SEARCH→TRACK transition fires only when it crosses τ sustained over the gating window. The gate is the difference between an autonomous system and a twitchy one.

03 · System design

Responsibility boundary & dataflow

The autopilot owns the inner loop and the tilt transition. ROS 2 owns the outer loop. The companion computer is never in the safety-critical path — any loss degrades to an autopilot-handled safe state.

04 · Engineering decisions

Decision log

Architecture decisions are recorded ADR-style — context, decision, rationale, status — and locked before code. The log is part of the deliverable: the point is showing the judgment, not just the result.

ADR-001 · PX4 over vendor-recommended ArduPlane

Existing PX4 fluency, native ROS 2 path, tiltrotor support via control allocation. Trade-off accepted and recorded: tiltrotor setup is hand-configured, not copy-paste from the vendor manual.

ADR-002 · One autopilot boundary, uXRCE-DDS primary

All autopilot I/O isolated in a single package; MAVLink behind the same interface as fallback. Swapping firmware touches one package, not the stack.

ADR-003 · Outer loop only in ROS 2

Attitude and tilt-transition control stay in firmware. The companion computer is never in the safety-critical loop; any loss degrades to autopilot RTL/loiter.

ADR-004 / 009 · Interfaces frozen first

Six-interface contract (4 msg, 2 srv) specified with frames and units per field, reviewed, and frozen 2026-05-31 — before any node was implemented.

ADR-005 · SITL-first, simulation parity

No code reaches the aircraft until it runs in PX4 SITL (Gazebo Harmonic, coastal world). Safety, cost, reproducibility.

ADR-006 · Edge inference on Hailo-8L

Pi 5 + AI HAT+ (13 TOPS) runs the detector onboard at ~3–4 TOPS/W — the transition decision needs no downlink. Thermal, 5 V-rail, and mass consequences flagged and tracked against the 2.5 kg MTOW.

ADR-007 · Firmware-agnostic VehicleState

The autopilot package re-publishes unified vehicle state; it is the sole consumer of PX4 message types among user packages.

ADR-008 · ENU/FLU everywhere; NED confined

All interfaces use REP-103 conventions; one package performs every frame conversion — a single point of truth for a classic source of integration bugs.

05 · Platform

The vehicle the autonomy flies on

Titan Dynamics Hornet — a 3D-printed (LW-PLA) 1.1 m tri-tiltrotor VTOL, two tilting front motors, fixed rear. Built and instrumented as the test platform. The airframe is the means; the autonomy is the contribution.

AIRFRAME

Wingspan 1100 mm · MTOW 2.5 kg
Cruise 50–75 kph · Max L/D 22
Stated efficiency 2.2 Wh/km

AVIONICS

Pixhawk 6C Mini · PX4
Pi 5 + AI HAT+ (Hailo-8L)
Pi Camera Module 3 · SiK 915 MHz

BINDING CONSTRAINT

Energy. Every onboard compute watt and payload gram trades against station time; guidance favours best-L/D loiter, and mass/power budgets are tracked against MTOW before flight.
